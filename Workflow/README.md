## Histórico / Legado

Documentos antigos, auditorias anteriores e relatórios de compatibilidade permanecem como histórico técnico. Eles não devem substituir a taxonomia atual.

Disgraceland não é mais operacional no vault. A lógica técnica aproveitada foi incorporada aos guias ativos e à taxonomia atual.

## Regra prática

Para novas migrações, usar esta ordem:

1. consultar a taxonomia atual;
2. consultar os guias ativos;
3. verificar relatórios recentes;
4. só então consultar histórico, se necessário.
